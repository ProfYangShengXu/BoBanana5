🔒 Bobanana 5.0 — 安全审计报告

审计日期: 2026-07-13 · 审计范围: 全量 Python 源码 · 审计角色: security-engineer

1. 审计摘要

对 bobanana4.0 代码库 14 个 Python 模块 + 3 个工具脚本进行安全审计。共发现 13 项安全问题，其中 1 严重、4 高、5 中、3 低。

⚠️ 总体风险评估：中
项目为本地运行的 CLI 工具，不涉及网络服务或外部用户，降低了远程攻击面。 但 os.system() 和大量 except: pass 构成了本地提权和可靠性风险。

2. 漏洞详情

CRITICAL V-01: os.system() 命令注入 (bobanana.py:142)
属性值
文件bobanana.py:142
代码os.system('python gui_dashboard.py')
风险os.system 通过 shell 执行命令。如果 gui_dashboard.py 路径被污染或当前目录有同名文件，可执行任意代码。
影响在 Windows 上，`%PATH%` 劫持可导致任意命令执行
修复替换为 subprocess.run([sys.executable, 'gui_dashboard.py'])
CVSS7.5 (AV:L/AC:H/PR:L/UI:R/S:U/C:H/I:H/A:H)
HIGH V-02: 裸 except: — 静默吞异常 (handoff_ticket.py:45,97)
文件行代码影响
handoff_ticket.py45except: (pass)FileNotFoundError 被静默忽略
handoff_ticket.py97except: (pass)JSON 解码错误被静默忽略

这些异常可能导致角色卡加载失败、工单丢失而不留任何痕迹。修复：改为精确异常类型 + logging.warning()。

HIGH V-03: 裸 except: — YAML 错误吞没 (role_tag_system.py:34)

except: 在 load_all_role_cards() 中捕获所有异常。如果 YAML 文件损坏，角色卡加载失败没有任何日志。

修复：捕获 yaml.YAMLError + FileNotFoundError，记录警告日志。

HIGH V-04: 裸 except: — JSON 状态丢失 (pipeline_orchestrator.py:222)

except: 在读取管线状态 JSON 时吞没所有异常。如果状态文件损坏，管线可能静默回退到空状态而不告警。

修复：捕获 json.JSONDecodeError + FileNotFoundError，回滚到上次有效状态并记录告警。

HIGH V-05: 裸 except: — HR 网络请求失败 (hr_recruitment.py:81)

except: 在 start_recruitment() 中吞没网络请求异常。如果 urllib.request 失败（网络不通、DNS 解析失败），用户得不到任何反馈。

修复：捕获 urllib.error.URLError + 超时异常，返回错误信息给调用方。

MEDIUM V-06: 裸 except: — 事件文件读取 (mcp_events.py:89)

except: 在 _check_file() 中吞没 JSON 解码错误。虽然已设 data = {} 作为 fallback，但无法区分"文件损坏"和"空文件"。

修复：捕获 json.JSONDecodeError，记录警告日志。

MEDIUM V-07: 裸 except: — 环境检查 (bobanana.py:39,45)

except: 在 check_env() 中 pip install 失败后无反馈。用户不知道依赖安装是否成功。

修复：检查 subprocess.run() 的 returncode，失败时输出提示。

MEDIUM V-08: MCP 路径遍历风险 (mcp_server.py)

handle_read_file 接收用户提供的 path 参数，未做路径消毒。理论上可通过 ../../../etc/passwd 读取任意文件。

但由于 MCP 仅用于本地 stdio 通信，攻击面有限。建议添加 os.path.abspath() 和 startswith 检查。

MEDIUM V-09: 硬编码路径 (多处)

多处代码直接硬编码 .reasonix/pipelines、.reasonix/state、skills/roles/ 等路径，未使用 os.path.join() 或配置常量。在非标准工作目录运行时可能出错。

MEDIUM V-10: GUI 启动器无超时/重试 (mcp_client.py)

connect() 的 timeout 参数在 direct 模式下未使用。subprocess 模式启动服务器无超时保护，服务器可能永久挂起。

修复：添加 subprocess.Popen 超时 + 指数退避重试。

LOW V-11: 无密钥/凭据管理

项目当前不涉及外部 API 密钥或数据库凭据，代码中无硬编码密码。但作为基础架构工具，建议预留 .env 或 keyring 支持。

LOW V-12: 日志与监控不足

项目使用 print() 而非 logging 模块。生产运行无法分级控制日志输出、无法通过日志文件排查问题。

LOW V-13: 无鉴权/加密 (MCP 协议)

MCP JSON-RPC 通过 stdio 通信，无身份验证或传输层加密。对于本地 CLI 工具，这是可接受的。如果未来增加远程 MCP 支持，必须添加 TLS + Token 鉴权。
