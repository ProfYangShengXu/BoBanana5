ADR-001: 管线存储迁移到工作区隔离

✅ 接受 | 2026-07-16

背景

当前管线状态全部存储在 .reasonix/ 下的固定路径中：

  • .reasonix/state/machine_state.json — 全局单例
  • .reasonix/pipelines/pl-*.json — 单个文件
  • .reasonix/cycle/ — 全局单例

当需要同时运行多条管线（如 Quant Kitset 管线 + 主开发管线）时，状态互相覆盖。

可选方案

方案描述优点缺点
A — 子目录隔离 ✅ .reasonix/pipelines/{workspace}/{pipeline_id}/ 每个管线独立子目录 完整隔离、便于调试、可扩展 需迁移旧数据
B — 文件名加前缀 pl-{workspace}-{timestamp}.json 改动最小 文件多了不易管理、状态机态文件仍混在一起
C — SQLite 数据库 单文件 SQLite 存所有管线 查询方便、ACID 引入新依赖、调试不方便（二进制）

决策

选择方案 A — 子目录隔离。理由：

  1. 纯文件系统方案，零新依赖
  2. 每个管线目录自包含：pipeline.json + machine_state.json + cycle_state.json
  3. 调试方便：直接 cat 或 vim 打开目录
  4. 与版本控制兼容：可单独 .gitignore
  5. 为未来并行管线提供天然沙箱

影响

  • 正面：管线完全隔离、支持并行、迁移路径清晰
  • 负面：需要修改 3 个文件的路径逻辑
  • 回滚：保留旧路径检测逻辑，新代码写新路径，旧代码仍可读旧路径
