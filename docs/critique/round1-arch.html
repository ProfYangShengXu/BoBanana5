🚨 架构批判 — Round 1/3

审查范围: pipeline_orchestrator.py, state_machine_engine.py, Bobanana.md

1. rounds 参数默认值不一致

init_pipeline() 默认 rounds=3，CLI 默认 rounds=1，cycle SKILL.md 又写死 3。同一个参数三个默认值，谁调谁不确定。应该统一为：函数默认值=1（安全），调用者显式传参。

2. advance_pipeline 多轮逻辑与正常流转耦合

CL通过后的多轮判断混在 advance_pipeline 的 terminal 处理代码里。这段逻辑本质上是"管线编排"而非"状态机流转"，应该抽到 cmd_advance 或一个新的 handle_post_transition 函数中。

3. cycle/state.json 的 phase 字段语义过载

phase 同时表示"状态机转换阶段"(boss-done)和"循环轮次阶段"(plan/exec)。cycle 的 plan/exec 和 SM 的 boss-done/arch-done 混在同一个字段里，后续解析时必须知道上下文才能正确理解。

4. 无测试覆盖多轮循环

68 个测试中没有一个测试多轮循环逻辑。新增的 round/max_rounds 字段、回 Boss 逻辑、next_prompt 写入都在 untested 状态下运行。

方案

  1. init_pipeline 默认 rounds=1
  2. 多轮判断从 advance_pipeline 抽出到独立函数
  3. cycle 用独立字段标记轮次，不和 SM phase 共用
  4. 新增多轮循环测试
