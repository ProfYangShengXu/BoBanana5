3. 通信架构

双向通信协议 
┌─────────────────┐        WebSocket         ┌──────────────────┐
│  Bobanana Python  │  JSON-RPC 2.0 over ws   │  Godot 4.x 实例  │
│  Pipeline Engine  │◄──────────────────────►│  (Headless/GUI)  │
│                   │   ws://localhost:9800    │                  │
│  tools.py         │                          │  addons/godox/   │
│  (bridge client)  │                          │  mcp_server.gd   │
└─────────────────┘                          └──────────────────┘

消息类型

方向消息说明
Agent → Godotscene.get_tree获取当前场景树
Agent → Godotscene.add_node添加节点到场景
Agent → Godotscene.set_property设置节点属性
Agent → Godotscene.run运行游戏
Agent → Godotinput.simulate模拟键盘/鼠标输入
Godot → Agentscene.updated场景变化事件通知
Godot → Agentlog.message实时日志推送
Godot → Agentgame.state游戏运行时状态

5. 插件增强设计

6. 角色卡设计原则

  • game-architect (OP)：游戏开发管线的架构师，类似 quant-architect 的格式
  • 策划类：偏文档/设计产出，不写代码。phase flag 用 design-done
  • 程序类：偏实现，写 GDScript/C#。phase flag 用 dev-done
  • 美术类：偏资源管理，不写代码。phase flag 用 art-done
  • 音频类：偏资源管理。phase flag 用 audio-done
  • 测试类：偏验证。phase flag 用 test-done
  • 格式：全部对齐 skills/roles/ 下的角色卡格式（role-card.yaml + SKILL.md + standards-brief.yaml）
