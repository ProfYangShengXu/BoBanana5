3. 接口定义

3.1 MCP 接口

MCP 客户端/服务器通过 JSON-RPC 2.0 通信，核心接口：

  • call_tool(name, arguments) → ToolResult
  • list_tools() → ToolDescription[]
  • subscribe_to_events(filters) → EventStream

3.2 状态机接口

  • StateMachineRuntime.load(config_path) → Runtime
  • Runtime.transition(phase, data) → (next_node, handoff_ticket)
  • Runtime.insert_temporary_node(node_def) → bool

3.3 角色卡接口

  • RoleCardRegistry.scan(path) → CardMetadata[]
  • RoleCardRegistry.validate(card_path) → ValidationResult
  • HandoffTicket.create(sender, receiver, artifacts) → Ticket

6. 状态机更新

当前状态机已在 state-machine.yaml 中定义，Boss→架构师→frontend-dev→client-dev→test-dev-engineer→security-engineer→fullstack-dev→架构师审核→终端。

本次架构设计后无需修改状态机拓扑，只需确认以下流转条件：

  • 架构师设计完成后 → frontend-dev（修复仪表盘 T-05/T-12/T-13）
  • frontend-dev 完成后 → client-dev（C++ 兼容性支持）
  • client-dev 完成后 → test-dev-engineer（T-06~T-11/T-14/T-15~T-17 新建测试）
  • test-dev-engineer 完成后 → security-engineer（安全审查）
  • security-engineer 完成后 → fullstack-dev（修复 except:pass T-01~T-04）
  • fullstack-dev 完成后 → 架构师验收 → 终端
